１. 网卡收到数据包，中断上半部处理

网络子系统的中断由硬件中断上半部和softirq下半部组成，这样分的原因在《understanding linux network internals》的chapter 9中有详细描述，简单的说就是网卡储存空间有限，在收到数据包后，需要尽快把数据拿到内存，因此讲这个处理放在中断处理上半部，此时禁止preempt。而当cpu闲暇时，再用ksoftirqd内核线程慢慢处理下半部。

NAPI

这里不得不提NAPI，NAPI是New API的缩写，它是一种相对于传统网卡driver的新的实现接口，主要是为提升performance考虑，非NAPI的驱动在将driver数据递交给内核时需要disable中断，而NAPI实现的驱动不需要，如果driver正在将队列中数据交给内核时又发生了新的中断，只需要中断上半部将网卡中的数据复制到driver的队列，当从中断上下文退出时，下半部继续取队列数据给内核，直到队列为空。如今大部分driver都是使用NAPI了。

alx　driver里附属于net_device的私有数据里，包含8个struct alx_napi结构的指针，真正的struct_npai结构在这里面，每个struct napi还对应了一个发送队列txq和一个接收队列rxq

２．softirq中断下半部处理

上半部最终抛出一个NET_RX_SOFTIRQ类型的softirq，实际上时告诉内核”hi, 紧急部分处理完成了，数据我已经放好了，你空了记得去处理”。什么时候空？这个是由该cpu上的ksoftirq线程决定的，这是一个优先级很低的内核线程，专门用于调度处理各个softirq(它也是每cpu一个)。
